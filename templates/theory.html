Lý thuyết & Ứng dụng Hash Table

1. Lý thuyết Hash Table
1.1. Nguyên lý hoạt động
  • Hash table là cấu trúc dữ liệu lưu trữ dạng key-value
  • Sử dụng hàm băm (hash function) để ánh xạ key thành index
  • Độ phức tạp trung bình: O(1) cho search/insert/delete
1.2. Các phương pháp xử lý collision

2. Ứng dụng trong AI/ML
2.1. Feature Hashing (Hashing Trick)
  • Chuyển đổi text thành vector số học
  • Giảm chiều dữ liệu mà vẫn giữ thông tin quan trọng
  • Ứng dụng: Text classification, Spam detection
2.2. Locality Sensitive Hashing (LSH)
  • Tìm kiếm các vector tương tự nhanh chóng
  • Ứng dụng: Image search, Document similarity
  • Độ phức tạp: O(1) thay vì O(n)
2.3. Bloom Filter
  • Kiểm tra membership với bộ nhớ tối thiểu
  • Ứng dụng: Cache filtering, Duplicate detection
  • False positive có thể chấp nhận được
3. Xử lý dữ liệu lớn
3.1. Distributed Hash Tables
  • Phân tán dữ liệu trên nhiều máy
  • Load balancing tự động
  • Fault tolerance cao
3.2. Memory Optimization
  • Compact hash table
  • Custom allocator
  • Memory pooling
3.3. Performance Tuning
  • Cache-friendly data layout
  • SIMD optimization
  • Lock-free concurrent access
